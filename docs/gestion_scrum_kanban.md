# Gestion Scrum du projet

## Contexte
Le projet de l’équipe 2 a été organisé avec une approche Scrum, tout en utilisant GitHub Projects en mode Kanban pour le suivi quotidien des tâches. L’objectif était de garder une vision claire de l’avancement, de limiter les blocages et de faciliter la collaboration entre les membres de l’équipe.

GitHub Projects a servi de tableau central pour visualiser les tâches, les priorités et l’état d’avancement. Les cartes ont été déplacées progressivement selon leur statut, ce qui a permis d’avoir un suivi simple et lisible :
- À faire
- En cours
- En revue / test
- Terminé

## Fonctionnement avec GitHub Projects en mode Kanban
Le projet a été piloté à travers un tableau Kanban GitHub, ce qui a permis de suivre les tâches de manière continue sans attendre la fin d’un sprint complet. Cette organisation a été utile pour :
- découper le travail en tâches plus petites et plus faciles à suivre ;
- visualiser immédiatement les blocages ou les tâches en attente ;
- répartir les actions selon la disponibilité de chacun ;
- garder une trace claire des décisions et de l’évolution du projet.

Chaque tâche a été déplacée selon son état d’avancement, ce qui a facilité les points d’équipe et permis d’identifier rapidement ce qui devait être repris, testé ou corrigé.

## Blocages rencontrés
Plusieurs blocages ont été rencontrés pendant le projet :

- **Coordination entre les membres** : il a parfois fallu synchroniser les travaux pour éviter les doublons ou les conflits de modifications.
- **Intégration frontend / backend** : certaines fonctionnalités nécessitaient un alignement précis entre les données exposées par l’API et les besoins de l’interface.
- **Gestion de l’authentification et des accès** : l’espace admin a demandé des ajustements pour sécuriser l’accès et rendre le flux d’utilisation clair.
- **Environnement technique** : le fonctionnement avec Docker, la base de données et les relances de services ont demandé de la vigilance.
- **Corrections de dernière minute** : comme dans beaucoup de projets agiles, certaines fonctionnalités ont demandé des retouches après les tests ou les retours d’utilisation.

Ces blocages ont été résolus progressivement grâce aux échanges dans l’équipe, au suivi dans GitHub Projects et à la répartition des responsabilités.

## Bilan
La gestion Scrum du projet a permis d’avoir une organisation claire, une meilleure visibilité sur l’avancement et une collaboration efficace. L’usage de GitHub Projects en mode Kanban a été particulièrement utile pour piloter les tâches en continu, suivre les priorités et réagir rapidement aux difficultés rencontrées.

Au final, cette méthode a renforcé la communication dans l’équipe et a aidé à livrer un projet plus structuré, avec des responsabilités bien identifiées.
